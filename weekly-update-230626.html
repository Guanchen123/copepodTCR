Algorithm assigning addresses not to peptides but to pairs / triples/ etc of peptides

  • All possible combinations of ITERS+1 out of N_POOLS

  • Then we need to order these combinations in a way that we can then produce actual addresses for peptide

    [0, 1, 2, 3, 4] - [0, 1, 2, 3] + [0, 1, 2, 4] [0, 1, 2, 4, 5] - [0, 1, 2, 4] + [0, 1, 2, 5] [0, 1, 2, 5, 6] - [0, 1, 2, 5] + [0, 1, 2, 6]

UP-and-DOWN algorithm

But the simplest way to do so I came up with is just order peptides addresses in a way that neighbours always produce together a combination of length 5.

So, I returned to my previous idea of ordering addresses in that way. Last time I did that, I encountered problems of too many collisions:

But this time I understood that we can simply remove the middle of the controversial region (the tip) which will improve the situation and at the same time will not spoil the order.

But when I did that another problem showed up. It turned out that similar addresses now have similar neighbours because of this UP-and-DOWN structure.

0 1 2 10
0 1 2 11
= 0 1 2 10 11

0 1 10 11
0 2 10 11
= 0 1 2 10 11

This problem can not be simply solved by removing one of the addresses because it would spoil the order. A bunch of addresses should be removed in this situation to get rid of this collision.

The order is now spoiled:
0 1 8 10
0 1 8 11
0 1 9 11
0 2 10 11
0 2 9 10
0 2 8 10

To restore the order, we need to remove addresses until neighbour addresses differentiate only by one amino acid:
0 1 8 10
0 1 8 11 - remove
0 1 9 11 - remove
0 2 10 11 - remove
0 2 9 10 - remove
0 2 8 10

Then:
0 1 8 10
0 2 8 10
together give us 0 1 2 8 10
We did not have this combination before, so we can leave it

Circular algorithm

Another way to assign addreses in a way that neighbours combined give length 5 is to imagine available pools not as a line but a circle.

And then we need to interate through this circle.

Properties of our algorithm

  • Decodable result â€” using pools pattern activation, we should understand, which two (or one, in case of the first and last peptides) led to this pattern
  • Robust scheme: ideally, one mistake still allows to find cognate peptide, but it would be alright, if one dropout leads to not more than 10 possible peptides with TRUE one
  • Flexible scheme: ideally, it should be applies to any number of peptides with overlap of any length
  • Number of pools should be minimized
  • Number of peptides in one pools should be minimized too (== number of pools to which one peptide goes should be minimized)
  • Number of peptides in pools should be more or less balanced
