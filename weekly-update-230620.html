Features

  • Decodable result — using pools pattern activation, we should understand, which two (or one, in case of the first and last peptides) led to this pattern
  • Robust scheme: ideally, one mistake still allows to find cognate peptide, but it would be alright, if one dropout leads to not more than 10 possible peptides with TRUE one
  • Flexible scheme: ideally, it should be applies to any number of peptides with overlap of any length
  • Number of pools should be minimized
  • Number of peptides in one pools should be minimized too (== number of pools to which one peptide goes should be minimized)
  • Number of peptides in pools should be more or less balanced

Ideas for the new algorithm

  1. Assign address not to a single peptide, but to a pair of peptides sharing the epitope (or N number of peptides sharing the epitope)
    • Flexible scheme: user can give us the list of peptides and number of aminoacids with which this peptide overlaps with neighbours, and we can assign addresses to N number of peptides with shared epitope (in our scheme - to pairs of peptides)
    • Absence of collisions, since we assign addresses to pairs of peptides, and these assigned addresses are unique
  2. Think not about the peptide, but about the overlapping epitope (similar to the first one)
    • Problem: what if peptides do not overlap at all?
  3. Divide all peptides into two groups of non-overlapping peptides and treat them as different samples
    • Limits our number of addresses

Error-correcting codes

Each address can be a message containing zeros and ones (zero for pool where peptides was not added and one for pool where in was added).

Address Message
[0, 1, 2, 3] 111100000000
[1, 4, 7, 9] 010010010100

And because of that we can use Hamming’s error-correcting method.

Number of parity bits: 2^p ≥ d + p + 1, where p is the number of parity bits and d is the number of data bits. Then we can use 11 data bits and 4 parity bits. Each value with only a single binary 1 and the remaining digits 0s (powers of 2) become parity bits. All other bits are data bits. Each parity bit covers all data bits that have the parity bit's value set to 1.

P1 checks D1, D3, D5, D7, D9, D11, D13, D15 P2 checks D2, D3, D6, D7, D10, D11, D14, D15 P4 checks D4, D5, D6, D7, D12, D13, D14, D15 P8 checks D8, D9, D10, D11, D12, D13, D14, D15

Parity bit is 0 if number of 1 in its data bits is even and 1 if number of 1 in its data bits is odd. After the error occurs, first all parity bits should be checked and changed accordingly. Then all parity bits are extracted and written in a reverse order. The number which is decimal version of this number is number of a data bit where the error occured.

Address Message Dropped pool Corrected message Found error
[6, 8, 9, 10, 14] 0000001011100010 6 110100001110001 6
[6, 10, 12, 13, 14] 0000001000101110 14 110100110010110 14

Problems:

  • there are very limited number of addresses that are suited to the composition of message decoded with error-correcting code (out of 3003 combinations of 5 out of 15, only 168 are suitable for us)
  • the balance of number of peptides in one pool is disturbed because there are pools (which are also parity bits) that will have more peptides
  • what if error happens in parity bit pool?

Decision: We will not use error-correcting codes in their original version. But we can use similar idea for our own algorithm because we do not need so high precision.
